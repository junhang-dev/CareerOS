# CareerOS: Career Branding Studio

## 한 줄 정의

CareerOS는 LinkedIn, 리멤버, GitHub, 포트폴리오, PDF 이력서를 대체하는 채용 플랫폼이 아니라,
여러 외부 채널에 보여줄 "나"를 정리하고 재가공하는 내부 기록장이다.

## 제품 관점

외부 채널의 역할:

- LinkedIn: 해외 네트워킹, 공개 프로필, 채용 노출
- 리멤버: 국내 경력직 프로필, 리크루터 접점
- GitHub: 코드와 프로젝트 증거
- 포트폴리오 페이지: 프로젝트 스토리와 시각적 신뢰
- 국문 경력기술서 / 영문 Resume: 최종 제출 문서

CareerOS의 역할:

- 흩어진 경력 자료, 프로젝트, 자격증, 논문, 공고 분석을 내부 자산으로 축적한다.
- 같은 경험을 직무와 채널에 맞게 다르게 번역한다.
- 지원 자동화보다 자기 이해, 포지셔닝, 문서 초안 생성을 우선한다.

## 핵심 철학

1. 외부 제출보다 내부 정리가 먼저다.

   좋은 이력서와 프로필은 지원 버튼을 많이 누르는 것보다, 내가 어떤 문제를 반복해서 풀어왔는지
   정확히 아는 데서 나온다.

2. 경험은 하나지만 내러티브는 여러 개다.

   예를 들어 `AI 비전 기반 RT 이미지 자동 판독 시스템`은 기아 Data Scientist 지원에서는
   물리 시스템 데이터 해석과 시계열/AI 경험이고, 현대 AX PM 지원에서는 제조 AX 과제 기획,
   외부 협업, 성과 관리 경험이다.

3. 모든 주장은 증거와 연결되어야 한다.

   프로젝트 문서, 이력서, 논문, 자격증, 공고 분석이 각각 근거가 된다. CareerOS는 이 근거를
   모아 채널별 문장으로 바꾸는 편집기다.

4. 자동 제출은 제품의 중심이 아니다.

   제출은 LinkedIn, 리멤버, 채용 홈페이지에서 하면 된다. CareerOS는 그 전에 어떤 나를 보여줄지
   결정하는 작업대다.

## 현재 우선 사용자

- 핵심 배경: 제조/정유 도메인 데이터, AI/ML 프로젝트 리딩, 공학 기반 문제 해결
- 현재 타깃:
  - 국내/해외 Data Analyst / Data Scientist
  - Machine Learning Engineer
  - 제조·모빌리티 AX / AI 직무
  - AX Project Manager

## CareerOS 안에서 관리할 자산 단위

- Career Evidence
  - 경력, 프로젝트, 수치 성과, 기술 스택, 증빙 파일
- Career Narrative
  - 나는 어떤 문제를 풀어왔는가
  - 어떤 산업과 직무로 확장 가능한가
  - 같은 경험을 어떤 언어로 다르게 설명할 것인가
- Channel Output
  - 국문 경력기술서
  - 영문 Resume
  - LinkedIn About / Experience
  - 리멤버 경력 소개
  - GitHub README
  - 포트폴리오 프로젝트 페이지
  - 지원서 문항 답변

## 지금까지의 제품 결론

CareerOS는 "채용공고를 모으는 앱"보다 "내 커리어를 증거 기반으로 정리하고, 채널별로 재가공하는
개인 브랜딩 엔진"에 가까워야 한다.

따라서 다음 개발은 더 많은 채용 플랫폼 기능보다, 현재 입력된 경력 데이터를 바탕으로 다음 산출물을
만드는 방향이 좋다.

- 국문 경력기술서 v1
- 영문 Resume v1
- LinkedIn / 리멤버 프로필 문구
- GitHub / 포트폴리오 프로젝트 문구
- 공고별 자기소개서/지원서 문항 초안
