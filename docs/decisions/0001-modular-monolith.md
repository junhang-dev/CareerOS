# ADR 0001: 모듈러 모놀리식으로 시작한다

## 상태

제안

## 결정

CareerOS MVP는 모듈러 모놀리식 구조로 시작한다.

## 이유

- 저장소가 비어 있는 상태에서 가장 빠르게 제품 흐름을 연결할 수 있다.
- 도메인 경계를 먼저 세우면 이후 워커/API 분리 비용을 줄일 수 있다.
- 싱글유저 MVP에서는 운영 복잡도를 낮추는 편이 유리하다.

## 결과

- 배포는 단일 앱 중심으로 간다.
- 외부 연동, 워커, 보안은 코드 경계로 먼저 분리한다.
- 고부하 구간이 확인되면 그때 배포 단위를 분리한다.
